# 🤖 ربات تلگرام مدیریت پنل 3X-UI

ربات تلگرامی کامل برای مدیریت پنل **3X-UI** به زبان فارسی.  
پوشش **۱۰۰٪ API رسمی** (۸۷ endpoint) + فرم‌های تعاملی قدم‌به‌قدم.

---

## ✨ امکانات کلی

| بخش | امکانات |
|-----|---------|
| 👑 **پنل ادمین** | مدیریت کامل سرور، اینباندها، کلاینت‌ها، نودها، تنظیمات |
| 👤 **پنل مشتری** | وضعیت اشتراک، خرید، QR کد، لینک اتصال |
| 🛒 **سیستم فروش** | تعرفه‌ها، پرداخت کارت‌به‌کارت، تأیید رسید، تمدید هوشمند |
| 🎯 **فرم تعاملی** | ساخت اینباند/کلاینت/تعرفه/نود بدون نیاز به JSON |
| 🔐 **امنیت** | ALLOWED_ADMINS، rate-limit، sanitize ورودی، لاگ کامل |

## 🚀 راه‌اندازی سریع

### پیش‌نیازها
- Python 3.10+
- پنل 3X-UI نصب‌شده و در دسترس
- توکن ربات از [@BotFather](https://t.me/BotFather)

### نصب

```bash
# کلون پروژه
git clone <repo-url>
cd 3xui-telegram-bot

# نصب وابستگی‌ها
pip install -r requirements.txt

# ساخت فایل تنظیمات
cp .env.example .env
nano .env

# اجرا
python bot.py
```

### فایل `.env`

```env
# توکن ربات از @BotFather
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ

# شماره کارت برای واریز (بدون فاصله)
CARD_NUMBER=6037991234567890

# نام صاحب کارت
CARD_OWNER=علی محمدی

# chat_id ادمین‌های مجاز — با کاما جدا کنید
# برای دریافت chat_id خود به @userinfobot پیام دهید
ALLOWED_ADMINS=123456789,987654321
```

> ⚠️ اگر `ALLOWED_ADMINS` خالی باشد، هیچ‌کس نمی‌تواند ادمین شود.

## ⚙️ تنظیمات اولیه در تلگرام

1. `/start` ← **ورود ادمین**
2. `/setup` ← آدرس پنل — مثال: `http://1.2.3.4:54321`
3. نام کاربری و رمز عبور پنل را وارد کنید
4. `/setcard` ← شماره کارت و نام صاحب کارت
5. منوی **🛒 فروش** ← تعرفه‌های خود را تعریف کنید

|-------|-------|

## 👑 امکانات پنل ادمین

### 📊 وضعیت سرور
- CPU، RAM، آپتایم، ترافیک لحظه‌ای و کل
- وضعیت و نسخه Xray
- دکمه‌های سریع: لاگ، ری‌استارت، توقف، آپدیت

### 🛠 مدیریت اینباندها — فرم تعاملی

به جای وارد کردن JSON، ربات قدم‌به‌قدم می‌پرسد:

```
① نام اینباند       → VPN-Main
② پورت             → 443
③ پروتکل           → [VLESS] [VMess] [Trojan] [Shadowsocks] ...
④ نوع شبکه         → [TCP] [WebSocket] [gRPC] [HTTPUpgrade] ...
⑤ امنیت            → [Reality] [TLS] [None]

اگر Reality انتخاب شد:
⑥ دامنه هدف        → yahoo.com:443
⑦ ServerNames      → yahoo.com,www.yahoo.com
⑧ Private Key      → (از دکمه X25519 جدید بگیرید)
⑨ ShortId          → abcd1234

اگر TLS انتخاب شد:
⑥ SNI              → example.com

اگر WebSocket انتخاب شد:
← Path             → /vless

⑩ حجم کل (GB)      → 0 (نامحدود)
⑪ انقضا (روز)      → 0 (نامحدود)

✅ اینباند ساخته شد!
```

سایر عملیات اینباند:

| عملیات | توضیح |
|---------|-------|
| 📥 ایمپورت | ایمپورت اینباند از JSON |
| ✏️ ویرایش | ویرایش تنظیمات با JSON |
| 🗑 حذف | حذف اینباند با تأیید |
| 🔛 فعال/غیرفعال | toggle وضعیت |
| 🔄 ریست ترافیک | ریست ترافیک یک اینباند |
| 🗑 حذف همه کلاینت‌ها | حذف دسته‌جمعی |
| 🔄 ریست همه اینباندها | ریست کل |
| 🔗 Fallbacks | مشاهده و تنظیم fallback |

### 👥 مدیریت کلاینت‌ها — فرم تعاملی

```
① نام کاربری      → ali          (فقط انگلیسی، بدون فاصله)
② حجم GB          → 50
③ روز انقضا       → 30
④ محدودیت IP      → 2
⑤ Telegram ID     → 0 (اختیاری)

✅ کلاینت ساخته شد!
📧 ایمیل نهایی: ali@123456789   ← ربات خودش می‌سازد
```

> 💡 ایمیل به فرمت `نام@آیدی‌عددی‌تلگرام` ساخته می‌شود تا دو کاربر
> همنام هرگز با هم تداخل نداشته باشند.

| عملیات | توضیح |
|---------|-------|
| 📋 لیست همه | لیست کامل با ترافیک و انقضا |
| 🔍 جستجو | جستجو با ایمیل + IP‌های متصل |
| ✏️ آپدیت | ویرایش حجم، انقضا، وضعیت |
| 🗑 حذف | حذف با تأیید |
| 🔄 ریست ترافیک | ریست مصرف |
| ♻️ ریست همه | ریست مصرف همه |
| 🧹 حذف منقضی‌ها | حذف خودکار |
| 🚫 پاک IP ها | پاک کردن لیست IP |
| 🔗 لینک‌های اتصال | vless/vmess/trojan/... |
| ⏰ آخرین آنلاین | زمان آخرین اتصال |
| 📦 Bulk Adjust | تمدید/اضافه حجم دسته‌جمعی |

## 🛒 جریان خرید / تمدید

```
مشتری «خرید / تمدید» را می‌زند
        ↓
تعرفه‌ها نمایش داده می‌شوند
        ↓
شماره کارت + مبلغ نشان داده می‌شود
        ↓
مشتری واریز کرده و تصویر رسید آپلود می‌کند
        ↓
ادمین‌ها نوتیف + رسید + دکمه ✅/❌ دریافت می‌کنند
        ↓
ادمین تأیید می‌کند ← سیستم هوشمند:
  • کلاینت موجود  ← تمدید از ته‌مانده انقضای فعلی
  • کلاینت جدید   ← ساخت اتوماتیک در پنل
        ↓
مشتری نوتیف + لینک اشتراک دریافت می‌کند 🎉
```

## 🎯 فرم‌های تعاملی

تمام عملیات مهم بدون نیاز به JSON، با فرم قدم‌به‌قدم:

| فرم | قدم‌ها | کاربرد |
|-----|--------|--------|
| `inbound` | ۷–۱۳ | ساخت اینباند (قدم‌های شرطی بر اساس پروتکل/امنیت) |
| `client` | ۵ | ساخت کلاینت — ایمیل یکتا `نام@tgid` اتوماتیک |
| `client_edit` | ۳ | ویرایش کلاینت موجود |
| `plan` | ۵ | تعریف تعرفه فروش |
| `bulk_adjust` | ۳ | تمدید/اضافه حجم دسته‌جمعی |
| `node` | ۷ | افزودن نود جدید |
| `card` | ۲ | تنظیم کارت بانکی |
| `change_user` | ۴ | تغییر یوزر/پسورد پنل |
| `xray_install` | ۱ | نصب نسخه Xray |
| `token` | ۱ | ساخت API Token |
| `group` | ۱ | ساخت گروه جدید |
| `group_rename` | ۲ | تغییر نام گروه |

**ویژگی‌های موتور فرم:**
- نوار پیشرفت `▓▓▓░░░ 3/6` بالای هر سوال
- دکمه‌های inline برای گزینه‌ها
- قدم‌های شرطی (مثلاً تنظیمات Reality فقط وقتی Reality انتخاب شده)
- مقادیر پیش‌فرض با Enter
- دکمه ❌ لغو در هر مرحله
- validation با پیام خطای فارسی

## 🔐 امنیت

### ALLOWED_ADMINS

```env
ALLOWED_ADMINS=123456789
```

فقط این chat_id ها می‌توانند `/setup` بزنند. اگر خالی باشد **هیچ‌کس** نمی‌تواند ادمین شود.  
chat_id خود را از [@userinfobot](https://t.me/userinfobot) بگیرید.

### سایر محافظت‌ها

| محافظت | توضیح |
|--------|-------|
| بررسی دوگانه ادمین | هر callback ادمینی قبل از اجرا `is_admin()` چک می‌شود |
| Sanitize ورودی | نام کاربری: فقط `a-z`, `0-9`, `_`, `-` — فارسی و فاصله رد می‌شوند |
| Rate-limit رسید | حداقل ۶۰ ثانیه بین هر ارسال رسید |
| محدودیت سفارش | حداکثر ۳ سفارش pending همزمان |
| UUID validation | در عملیات حذف کلاینت UUID اعتبارسنجی می‌شود |
| لاگ کامل | همه عملیات حساس در `bot.log` ثبت می‌شوند |
